### 微速滚动晃动 — 多轮排查最终根因

| 盲点 | 根因 | 修复 |
|:---|:---|:---|
| displayScrollLine 浮点 roundtrip | `floor(localScrollTop/scaleFactor/28)` 不精确 | `displayScrollTopRef` 存 DOM 层 scrollTop，`floor(displayScrollTopRef/28)` |
| ResizeObserver 强制回流 | 滚动中 ResizeObserver 频繁回调 → `clientHeight` 读取 → layout thrashing | **删除 ResizeObserver**，改为 SearchPanel emit heightChange → 显式调用 updateContainerHeight |
| BlockCacheManager 每帧重建数组 | `loadBlocks` 即使 blocks 全部 loaded 也调用 `onBlocksChanged` → `visibleNodes=[...nodes]` → Vue 响应式触发全量重渲染 | `if (needsUpdate) onBlocksChanged` |
| handleNeedNodes 重复请求 | 同 block range 每帧触发 loadBlocks | `_lastReqStartBlock`/`_lastReqEndBlock` 守卫 |
| .tree-nodes 无 GPU 加速 | position:absolute+transform 未提升到合成层 | `will-change: transform` |

## 明确的结论（下次不再重复验证）

1. **ResizeObserver 在虚拟滚动场景下有害**——频繁触发强制回流，已删除。容器高度变化通过显式调用 `updateContainerHeight()` 解决。
2. **布尔标志 `isWheelSource` 有单次消费缺陷**——同手势多帧只第一帧有效。改用 `lastWheelTime` 时间戳 + 150ms 窗口。
3. **`lastRenderedLine` 解耦必须保持状态机完整**——只抑制渲染 emit，状态机/timing/RAF 永远执行。
4. **`displayScrollLine` 绝不能用 `localScrollTop/scaleFactor` 链**——浮点积差。只能用 DOM 层 `scrollTop` 直接计算。
5. **BlockCacheManager `onBlocksChanged` 必须在 `needsUpdate=true` 时才调用**——否则每帧重建 3000 元素数组触发 Vue 全量重渲染。
6. **Vue scoped CSS 不支持 v-html 注入元素**——需要全局 `<style>` 块覆盖。
7. **`watch(props.jumpToLine)` 值不变时 Vue 不触发**——需要 `jumpTrigger` 计数器。

## 决策记录

- **骨架屏设计**：P0 状态机(IDLE→SCROLLING_FAST→RENDERING→READY) + RAF 驱动 + 双重输入判断

- **容器高度同步**：ResizeObserver 删除 → 显式调用(避免强制回流)
